core¶

The core module contains the following submodules:
  • core.components

core.components¶

The core.components module contains:

  • The core classes which represent the main game model:

    • core.components.Game, which represents the structure of the game (interactive novel or text adventure).
    • core.components.Resource, which represents global information, such as the date or the weather in the fictional world, or values like the health state of the player.
    • core.components.Scene, which represents a location or a state in the fictional world.
    • core.components.Action, which represents an action that can be triggered by the player. A core.components.Action consists of a core.components.Condition and an core.components.Effect. The core.components.Effect is only resolved if the condition is True.
    • core.components.Condition, which represents a logical evaluation which can be part of an core.components.Action. When evaluated it returns a value of True or False.
    • core.components.Effect, which represents the outcome of a given Action that the player performs. An core.components.Effect can be anything from changing the scene, interacting with the world or even losing the game.

  • The core.components.customisable module which contains the specific functions used in the core.components.Effect and core.components.Condition classes.

class core.components.Action(name='default action', visible=True, enabled=True, active=True, effectsIfTrue=[], effectsIfFalse=[], conditions=[])[source]¶

The Action class represents an action that either occurs by itself or can be triggered by the player. An action has three switches which can be toggled: - Visible: if an action is visible it means that the player can see it as an available action. - Enabled: if an action is enabled it means that it can be viewed and triggered. - Active: if an action is active it means that it needs to be triggered by the player, otherwise it is performed on every frame.

Attributes :
  • name: A string containing the name of the action (e.g. “Go south”).
  • conditions: A list containing the Condition objects that need to be evaluated before the action is performed.
  • effectsIfTrue: A list containing the Effect objects that will be resolved if all of the conditions return True.
  • effectsIfFalse: A list containing the Effect objects that will be resolved if all of the conditions do NOT return True.
  • visible: A bool representing whether the action is visible or not.
  • enabled: A bool representing whether the action is enabled or not.
  • active: A bool representing whether the action is active or not.
conditionsAreTrue()[source]¶

Checks whether all of the Condition objects in condition are evaluated as True.

Returns :
  • True if all conditions are evaluated as True.
  • False otherwise.
perform()[source]¶

Resolves the Effect objects from effectsIfTrue if conditionsAreTrue() returns True. Resolves the Effect objects from effectsIfFalse if conditionsAreTrue() returns ``False`.

class core.components.Condition(conditionFunction=None, *args)[source]¶

The Condition class is used in an Action object to create conditional evaluation of actions.

Attributes :
  • conditionFunction: A function which is used as an operator to compare args.
  • args: A list containing the arguments for the conditional (usually a left-hand side and a right-hand side element).
  • rawArgs: A list containing the names of the arguments from args.
  • evalArgs: A list which is populated with the actual values of args, which are evaluated during run-time.
evaluate()[source]¶

Evaluate the condition.

Returns :
  • A bool.
class core.components.Effect(effectFunction=None, *args)[source]¶

The Effect class is used in an Action object to create the output of actions.

Attributes :
  • effectFunction: A function which performs the wanted behaviour of the effect.
  • args: A list containing the arguments which are passed to effectFunction.
  • rawArgs: A list containing the names of the arguments from args.
  • evalArgs: A list which is populated with the actual values of args, which are evaluated during run-time.
  • parent: A container of the Action object to which the current Effect objects belongs.
resolve()[source]¶

Resolve the effect.

class core.components.Game[source]¶

The Game class basically represents the structure of the game (interactive novel or text adventure). Therefore it contains the various Scene, global Resource and global Action instances of the game. It also keeps track of the starting scene and the current scene.

Attributes :
  • name: A string with the name of the game.
  • author: string with the author(s) of the game.
  • startingScene: A Scene object which is the first scene that the game will load.
  • currentScene: A Scene object which is the scene the player is currently in.
  • scenes: A list of Scene objects which holds all of the scenes of the game, along with their corresponding attributes.
  • globalResources: A list of Resource for storing global resources, such as player HP.
  • globalActions: A list of Action for storing global actions, such as actions which are always available to the player (e.g. ‘help’) or actions such as checking if the player’s HP falls under a certain threshold.
  • narrative: A list of narrative elements (e.g. a string equal to “You go south.”).
addResource(name, value)[source]¶

Initialise a Resource object and add it to the game’s Global Resources list globalResources.

Parameters :
  • name: The name of the new resource.
  • value: The value of the new resource.
getResourceByName(name)[source]¶

Finds and returns a Resource object given a string with the name of the resource. It returns the first resource which matches the name, starting by searching through the current scene and then through the global resources.

Parameters :
  • name: A string containing the name of the resource.
Raises :
  • “ERROR: Resource not found.” if no Resource which matches name is found.
Returns :
  • The first Resource object in globalResources with a name that matches name.
getSceneByName(name)[source]¶

Finds and returns a Scene object given a string with the name of the scene.

Parameters :
  • name: A string containing the name of the scene.
Raises :
  • “ERROR: Scene not found.” if no Scene which matches name is found.
Returns :
  • The first Scene object in scenes with a name that matches name.
removeResource(resource)[source]¶

Remove a Resource object from the the game’s Global Resources list globalResources.

Parameters :
  • resource: The Resource object to be removed.
class core.components.Resource(name, value)[source]¶

The Resource class represents global information, such as the date or the weather in the fictional world, or values like the health state of the player. It is basically a tuple, containing a name attribute and a value attribute.

Attributes :
  • name: A string which holds the name of the Resource, e.g. ‘HP’.
  • value: This attribute holds the actual value of the resource, which can be of any type. For example, if the resource represents the player’s HP, the value of the Resource can be an int of value 10.
class core.components.Scene(description='default description', name='default room')[source]¶

The Scene class represents a location or a state in the game. Just like a Game object contains lists of global resources and global actions, a Scene object contains lists of local resources and local actions.

Attributes :
  • name: A string containing the name of the scene (e.g. “Cave 1”).
  • description: A string containing the description of the scene, which is used in the narrative of the game (e.g. “There are two doors in front of you”.)
  • resources: A list of Resource objects known as local resources of a scene.
  • actions: A list of Action objects known as local actions of a scene.

core.components.customisable¶

The core.components.customisable module is the layer which can be modified by users during the development of games or the development of a game editor.

It contains the following modules:
  • core.components.customisable.effects, where the functions used in core.components.Effect objects are stored.
  • core.components.customisable.conditions, where the functions used in core.components.Condition objects are stored.
class core.components.customisable.effects.EffectFunctions[source]¶

This class contains all the customisable functions which are used to represent the output of an action e.g. go to another scene or give information to the player. The effectDict container is a dictionary containing key-value pairs of a function (e.g. goToScene()) and its name in a human-readable form, for use in an editor or elsewhere (e.g. ‘Go to scene’)

class core.components.customisable.conditions.ConditionFunctions[source]¶

This class contains all the customisable functions or operators which are used to represent the conditions of an action e.g. whether a core.components.Resource has a value of True. The conditionDict container is a dictionary containing key-value pairs of an operator/function (e.g. operator.eq()) and its name in a human-readable form, for use in an editor or elsewhere (e.g. ‘equals’).

It also contains functions used to interface with the editor (the default editor is written in C#):
  • getEffectCount(), returns the count of available functions in core.components.customisable.effects.EffectFunctions.
  • getConditionCount(), returns the count of available functions in core.components.customisable.conditions.ConditionFunctions.
  • getEffectKeys(), returns the keys of available functions in core.components.customisable.effects.EffectFunctions.
  • getConditionKeys(), returns the keys of available functions in core.components.customisable.conditions.ConditionFunctions.
